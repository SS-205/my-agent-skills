# Git Commit Message 生成器

## 概述

分析 Git 仓库中的代码变更（Diff），自动提取变更类型与功能模块，生成符合团队统一格式规范 `AAAAA：【BBBB】CCCCCC` 的 Commit Message。

## 触发条件

当用户请求生成提交信息时触发，例如：
- "帮我生成 commit message"
- "写一个 commit"
- "根据当前改动生成 commit 提交信息"
- "生成提交日志"

---

## 工作流程

### 步骤 1：获取 Git 变更信息

执行以下命令获取变更内容：

```
git diff --stat
```

获取文件变更概览。然后根据文件范围，读取相关文件的变更内容：

```
git diff
```

或查看暂存区变更：

```
git diff --cached
```

### 步骤 2：分析变更内容

分析变更文件的类型、范围和具体修改内容，重点关注：
- 新增/修改了哪些功能模块
- 修复了什么问题
- 重构了什么逻辑
- 修改了哪些文档或配置
- 变更涉及的功能领域

### 步骤 3：确定 commit type

根据变更内容，从以下类型中选择最匹配的一个：

| Type | 含义 | 适用场景 |
|------|------|---------|
| Feature | 添加新功能或特性 | 新增功能、新的接口、新的 UI 页面等 |
| Fix | 修复 bug 或问题 | 修复程序缺陷、逻辑错误（如果发现 RDM bug 编号，应附加在描述中） |
| Refactor | 重构代码结构或逻辑 | 代码重构、设计模式优化、模块拆分合并等 |
| Docs | 更新文档 | 注释、README、API 文档、设计文档等 |
| Style | 修改代码风格或格式 | 格式化、缩进、重命名变量、删除多余空行等（不改变逻辑） |
| Test | 增加或修改测试用例 | 新增单元测试、集成测试、修改测试用例等 |
| Optimize | 优化性能或效率 | 性能优化、内存优化、网络请求优化、数据库查询优化等 |
| Merge | 合并分支或变更 | 分支合并、代码同步等 |

### 步骤 4：确定 feature 名称（B）

根据变更涉及的功能模块，提取中文功能名称。例如：
- "登录模块"
- "订单列表"
- "用户权限"
- "数据导出"
- "消息推送"

### 步骤 5：生成 commit message

严格按照以下格式生成：

```
AAAAA：【BBBB】CCCCCC
```

其中：
- **AAAAA**：步骤 3 确定的 commit type 英文名称（首字母大写）
- **BBBB**：步骤 4 确定的功能中文名
- **CCCCCC**：具体修改内容的简要描述（中文），通常 10-30 字

### 示例

| 变更内容 | 生成的 commit message |
|---------|----------------------|
| 新增用户登录功能 | `Feature：【登录模块】新增用户手机号登录功能` |
| 修复订单列表空指针异常（RDM #1234） | `Fix：【订单列表】修复空指针异常导致页面崩溃（RDM #1234）` |
| 优化首页图片加载速度 | `Optimize：【首页】优化图片懒加载策略，减少首屏加载时间` |
| 重构用户权限校验逻辑 | `Refactor：【权限管理】重构权限校验中间件，简化判断逻辑` |
| 合并 feature/login 分支到 develop | `Merge：【登录模块】合并 feature/login 分支到 develop` |
| 更新 API 文档 | `Docs：【API文档】更新用户接口的请求参数说明` |
| 格式化代码风格 | `Style：【全局】统一使用 Kotlin 官方代码风格` |
| 添加订单模块单元测试 | `Test：【订单模块】为订单计算逻辑添加单元测试覆盖` |

## 注意事项

- **AAAAA** 部分使用英文，首字母大写
- **BBBB** 使用中文，用全角括号【】包裹
- **CCCCCC** 使用中文描述，简洁明了
- 如果是 Fix 类型，应检查是否有关联的 RDM bug 编号，如有则追加在描述末尾（如 `（RDM #1234）`）
- 如果变更涉及多个类型，选择最主要的一个
- 如果涉及多个功能模块，选择主要的模块名称
